## 1. Philosophie

ThorV4 est une réécriture en C++17 visant à être **la version la plus simple possible**
d'un contrôleur de WorkCell robotisé, tout en restant fonctionnellement équivalente
à ThorV2.

**Règles absolues (cf. AGENTS.md) :**
- KISS : une fonction = une responsabilité, ~40 lignes max
- Zéro template dans notre code
- Zéro lambda dans notre code
- Header-first : `.h` d'abord, `.cpp` ensuite
- Doxygen sur tout
- Pas de `using namespace` dans les headers
- `camelCase` pour tout sauf classes/structs/fichiers (`PascalCase`)

**Principes tirés de V2 et V3 :**
- V2 (C) : fonctionnellement complet mais complexe (double processus, plugins .so, vtable C)
- V3 (C++) : bonnes idées (GalilProtocol pur, sandbox Lua, DH kinematics) mais trop de couches (12), singletons, namespaces profonds, `Result<T>` monadique
- V4 = ce que V3 aurait dû être : V2 fonctionnel + V3 propre − V3 complexe

**Flux de contrôle principal :**

```
Lua → Primitives (méthodes C++ des composants) → Driver → Hardware

Lua: robot:moveTo(100, 200, 50)
  → Robot::moveTo(x, y, z)
    → DHKinematics::inverseKinematics() → angles joints
    → conversion angles → counts (countsPerUnit)
    → m_controller->getDriver()->moveAbsolute(counts)
      → GalilProtocol::encodeMoveAbsolute(counts) → "PAA=...,B=...,C=..."
      → sendRaw("PAA=...,B=...,C=...") → TCP 2323
      → GalilProtocol::decodeReply(response)
      → GalilProtocol::encodeBeginMotion(axes) → "BGABC"
      → sendRaw("BGABC")
```

**Possède :** EventBus, LuaEngine, Workcell (unique_ptr).

**Dépendances :** Tous les autres modules qu'il instancie.

### 4.2 EventBus

**Fichiers :** `Include/Agent/EventBus.h`, `Src/Agent/EventBus.cpp`

**Rôle :** File d'événements lock-free, multi-producer, single-consumer.
Utilise l'algorithme Vyukov MPMC (sequence-lock par slot).
Seul mécanisme de communication entre le thread listener Galil et le thread principal.

**Priorités :** Emergency → Critical → High → Normal → Background

**Payloads typés (union) :** MotionEvent, SafetyEvent, IOEvent, ScriptEvent,
PrimitiveEvent, + un fallback générique (int + double + string[64]).

**Algorithme :**
- Ring buffer de 2048 slots par queue (puissance de 2)
- Chaque slot a un `atomic<uint64_t> sequence`
- Producer : CAS sur `tail`, attend que `sequence == tail`, écrit, `sequence = tail + 1`
- Consumer : lit si `sequence == head + 1`, consomme, `sequence = head + SIZE`
- Queue pleine → drop + compteur, pas de spin
- Latence de publication : ~20-50 ns (un CAS + une écriture)

**Pas de worker threads internes.** Le main loop consomme les événements.

```cpp
class EventBus {
public:
    bool publish(const Event& event);
    int consume(int maxPerPriority, std::vector<Event>& eventsOut);
    bool hasEmergencyEvents();

private:
    EventQueue m_emergencyQueue;   // 2048 slots
    EventQueue m_queues[5];        // Critical..Background, 2048 slots chacun
};
```

**Producteurs :** GalilDriver::listenLoop (thread), LuaEngine (main thread),
Agent lui-même, WebServer (threads CivetWeb, futur).

**Consommateur :** Agent::tick() (main thread uniquement).

### 4.3 LuaEngine

**Fichiers :** `Include/Agent/LuaEngine.h`, `Src/Agent/LuaEngine.cpp`

**Rôle :** Encapsule le `sol::state` (VM Lua 5.4.7 via sol2). Exécute les scripts,
gère les erreurs, fournit des fonctions globales à Lua.

**Sandbox :**
- Pas de `os`, `io`, `debug` (sandboxing)
- `SOL_ALL_SAFETIES_ON=1` (vérifications de types)
- Instruction hook pour interrompre les scripts bloquants (timeout)

```cpp
class LuaEngine {
public:
    bool initialize();
    bool loadScriptFile(const std::string& filePath);
    bool runString(const std::string& code);
    sol::state& state();
    std::string lastError() const;
    void setEventBus(EventBus* bus);

private:
    sol::state  m_lua;
    std::string m_lastError;
    EventBus*   m_eventBus;
};
```

**Fonctions globales enregistrées dans Lua :**
- `thor.log(level, message)` — log côté C++
- `thor.publish(eventType, message)` — publie un événement dans l'EventBus

**Possédé par :** Agent.

**Dépendances :** sol2 (thor_sol2), EventBus* (optionnel).

### 4.4 Workcell

**Fichiers :** `Include/Agent/Workcell.h`, `Src/Agent/Workcell.cpp`

**Rôle :** Charge le fichier WorkCell JSON, instancie les composants,
les enregistre dans Lua, et fait le câblage composants → contrôleurs.

```cpp
class Workcell {
public:
    // Existant (non modifié)
    bool loadFromFile(const std::string& filePath);
    Component* findComponent(const std::string& id) const;
    const std::vector<WorkcellComponentEntry>& getComponents() const;

    // Nouveau
    bool loadComponents(sol::state& lua);
    bool wireComponents(EventBus* bus);
    void startListeners();
    void shutdown();

private:
    std::vector<WorkcellComponentEntry> m_components;
    std::string m_lastError;
};
```

**loadComponents() :** Pour chaque entrée dans le JSON :
1. `createComponent(type)` (factory existante)
2. `instance->loadFromFile(config)` (existant)
3. `instance->registerInLua(lua)` (nouveau)

**wireComponents() :**
1. Pour les contrôleurs : instancier `GalilDriver`, appeler `driver.connect()`
2. Pour les autres composants : résoudre `entry.controller` → `findComponent(id)` → `entry.instance->setController(controller)`

**Possédé par :** Agent.

**Dépendances :** sol::state (pour registerInLua), EventBus* (transmis aux drivers).

### 4.7 GalilProtocol

**Fichiers :** `Include/Agent/Drivers/GalilProtocol.h`, `Src/Agent/Drivers/GalilProtocol.cpp`

**Rôle :** Codec pur pour le langage ASCII Galil DMC. Fonctions statiques,
zéro état, zéro I/O. Partagé entre tous les modèles Galil (8240, 2143, 4040...).

```cpp
class GalilProtocol {
public:
    // Encode (valeurs C++ → strings Galil)
    static std::string encodeMoveAbsolute(const std::vector<double>& counts);
    static std::string encodeMoveRelative(const std::vector<double>& deltas);
    static std::string encodeBeginMotion(const std::vector<int>& axisIndices);
    static std::string encodeStopAll();           // "ST"
    static std::string encodeMotorOff();          // "MO"
    static std::string encodeServoHere();         // "SH"
    static std::string encodeHome(int axisIndex);
    static std::string encodeTellPosition();      // "TP"
    static std::string encodeReportPosition();    // "RP"
    static std::string encodeSpeed(const std::vector<double>& speeds);
    static std::string encodeAcceleration(const std::vector<double>& accels);
    static std::string encodeDeceleration(const std::vector<double>& decels);

    // Decode (strings Galil → valeurs C++)
    static bool decodeReply(const std::string& raw,
                            std::string& payloadOut, bool& isErrorOut);
    static bool parseAxisValues(const std::string& payload,
                                std::vector<double>& valuesOut, int expectedCount);
};
```

**Séparation GalilProtocol / GalilDriver :**
- `GalilProtocol` = le langage Galil (quoi dire)
- `GalilDriver` = le transport TCP + séquencement (comment et quand le dire)
- Cette séparation est pour la **lisibilité** et la **réutilisation entre modèles**, pas pour swapper le protocole

**Possédé par :** personne (fonctions statiques).

**Dépendances :** aucune (que `std::string` et `std::vector`).

### 4.8 GalilDriver

**Fichiers :** `Include/Agent/Drivers/GalilDriver.h`, `Src/Agent/Drivers/GalilDriver.cpp`

**Rôle :** Driver TCP pour contrôleur Galil DMC. Possède deux sockets (2323 command,
2324 async). Utilise GalilProtocol pour encoder/décoder. Gère timeouts et erreurs.

```cpp
class GalilDriver {
public:
    bool connect(const std::string& address, int commandPort, int messagePort);
    void disconnect();
    bool isConnected() const;

    // Commandes motion
    bool moveAbsolute(const std::vector<double>& counts);
    bool moveRelative(const std::vector<double>& deltas);
    bool beginMotion(const std::vector<int>& axisIndices);
    bool stopAll();
    bool motorOff();
    bool servoHere();
    bool home(int axisIndex);
    bool setSpeeds(const std::vector<double>& speeds);
    bool setAccelerations(const std::vector<double>& accels);

    // Requêtes
    bool getPositions(std::vector<double>& positionsOut);
    bool getDigitalInput(int pin, bool& valueOut);
    bool setDigitalOutput(int pin, bool value);

    // Listener asynchrone (port 2324)
    void startListener(EventBus* bus);
    void stopListener();

    std::string lastError() const;

private:
    int  m_commandSocket;
    int  m_messageSocket;
    bool m_connected;

    std::thread m_listenerThread;
    bool m_listenerRunning;
    EventBus* m_eventBus;

    std::string m_lastError;

    bool sendRaw(const std::string& command, std::string& responseOut, int timeoutMs);
    void listenLoop();   // exécuté dans m_listenerThread, lit le port 2324,
                         // parse les messages async, publie dans EventBus
};
```

**Pas d'abstraction ITransport.** Le driver fait du TCP directement.

**Pas de MockTransport.** On utilise xmul (émulateur Galil externe) pour les tests.

**Possédé par :** Controller.

**Dépendances :** GalilProtocol, EventBus*, sockets POSIX.

### 4.10 DHKinematics

**Fichiers :** `Include/Agent/Kinematics/DHKinematics.h`, `Src/Agent/Kinematics/DHKinematics.cpp`

**Rôle :** Solveur cinématique générique basé sur les paramètres
Denavit-Hartenberg. Fonctionne pour tout robot série.

```cpp
class DHKinematics {
public:
    bool loadFromJson(const std::vector<DHParameter>& dhParams);

    bool inverseKinematics(double x, double y, double z,
                           std::vector<double>& jointAnglesOut);
    void forwardKinematics(const std::vector<double>& jointAngles,
                           double& xOut, double& yOut, double& zOut);

private:
    std::vector<DHParameter> m_dhParams;

};
```

**Algorithme IK :** Newton-Raphson avec amortissement Levenberg-Marquardt
pour gérer les singularités (amélioration par rapport à V2).

**Futur :** Si un robot nécessite une cinématique plus rapide, on pourra
ajouter une classe spécifique (ex: `ATMKinematics`) et le Robot choisira
laquelle utiliser selon le modèle.

## 5. Modèle de threading

```
┌─────────────────────────────────────────────────────────┐
│                    Threads ThorV4                        │
│                                                          │
│  Thread principal (Agent::run)                           │
│  ┌────────────────────────────────────────────────────┐ │
│  │  • Boucle événementielle                           │ │
│  │  • Consomme EventBus (single consumer)             │ │
│  │  • Exécute les scripts Lua (bloquant)              │ │
│  │  • Appelle les primitives (bloquant)               │ │
│  │  • Télémétrie périodique (~10 Hz)                  │ │
│  │  • Watchdog check                                  │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  Thread listener Galil (GalilDriver::listenLoop)        │
│  ┌────────────────────────────────────────────────────┐ │
│  │  • Lecture bloquante sur socket TCP 2324           │ │
│  │  • Parse messages async (MG, limits, erreurs)      │ │
│  │  • Publie dans EventBus (multi-producer)           │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  Threads CivetWeb (futur, gérés par la bibliothèque)    │
│  ┌────────────────────────────────────────────────────┐ │
│  │  • HTTP request handling                           │ │
│  │  • Lecture seule sur EventBus/Workcell             │ │
│  │  • Écritures sérialisées par mutex dans WebServer  │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**Comparaison V2 → V4 :**

| Thread | V2 | V4 | Justification |
|---|---|---|---|
| Main loop | ✓ (actif) | ✓ (actif) | Essentiel |
| EventBus workers | 4+1 | 0 | Main loop consomme directement |
| Safety monitoring | 1 (10kHz) | 0 | Intégré dans main loop |
| Feedback polling | 1 (100Hz) | 0 | Intégré dans main loop |
| Heartbeat | 1 | 0 | Intégré dans main loop (futur) |
| Telemetry | 1 | 0 | Intégré dans main loop |
| Galil listener | 0 (intégré) | 1 | Port 2324 est bloquant |
| Web server | 1 (Mongoose) | 1-2 (CivetWeb) | Bibliothèque externe |
| **Total** | **~11** | **~3** | Réduction de 70% |

**Règle :** Seul le thread principal modifie l'état des composants. Le listener
Galil est read-only + publication EventBus.

### ⚠️ Risque identifié : blocage du thread principal

Si un script Lua ou une primitive bloque (ex: `robot:moveTo()` de 30 secondes),
le heartbeat, la safety check, la télémétrie et le watchdog sont gelés.

**Mitigations existantes (sans threads supplémentaires) :**
- Le Galil a ses propres limites hardware (TL, ER, OE, FL/BL) indépendantes du heartbeat
- L'E-Stop câblé fonctionne hors software
- Le watchdog peut être OS-level (timerfd + `SCHED_RR`) plutôt qu'un thread

**Évolutions possibles (si le risque se matérialise) :**
- Dédier un thread léger « safety tick » (1 kHz, vérifie juste une variable atomique)
- Exécuter les primitives longues dans un thread worker séparé avec callback
- Timeout obligatoire sur toutes les primitives bloquantes

> **Décision :** On garde le modèle à 3 threads pour la V1. Si le heartbeat
> ou la safety sont impactés en pratique, on ajoutera un thread safety tick
> minimal (pas de refonte complète).

## 6. Gestion des erreurs

ThorV4 utilise le pattern `bool` + `lastError()` — pas de `Result<T>`, pas d'exceptions :

```cpp
bool Robot::moveTo(double x, double y, double z, double speed)
{
    if (!m_kinematics.inverseKinematics(x, y, z, m_targetJoints)) {
        m_lastError = "IK failed: position out of workspace";
        return false;
    }
    if (!m_controller->getDriver()->moveAbsolute(anglesToCounts(m_targetJoints))) {
        m_lastError = "Controller error: " + m_controller->getDriver()->lastError();
        return false;
    }
    return true;
}
```

Chaque niveau de la pile propage l'erreur avec un contexte utile.

## 7. Gestion des erreurs Lua

**Niveau 1 — Erreur de syntaxe/runtime :**
```cpp
try {
    m_lua.script_file(path);
} catch (const sol::error& e) {
    m_lastError = e.what();
    publish(ScriptError, e.what());
    return false;
}
```

**Niveau 2 — Timeout du script :**
Instruction hook Lua (LUA_MASKCOUNT) qui interrompt le script après N instructions.

**Niveau 3 — Erreur de primitive :**
Les primitives retournent un `bool`. Le script Lua peut vérifier :
```lua
local ok = robot:moveTo(100, 200, 50)
if not ok then
    thor.log("ERROR", robot:lastError())
    return
end
```

## 10. Comparaison V2/V3/V4

| Aspect | ThorV2 | ThorV3 | ThorV4 |
|---|---|---|---|
| **Langage** | C (C11) | C++20 | C++17 |
| **Processus** | 2 (Supervisor + Agent) | 1 (Agent seul) | 1 (Agent seul) |
| **Plugins** | Dynamiques (.so) | Dynamiques (.so) | Aucun (compilé en dur) |
| **Templates** | N/A (C) | 3 autorisés (`Result<T>`, etc.) | 0 |
| **Lambdas** | N/A (C) | Limitées | 0 |
| **EventBus** | CAS simple (faible) | Vyukov MPMC (correct) | Vyukov MPMC (correct) |
| **EventBus workers** | 4 + 1 emergency | 0 (externe) | 0 (main loop consomme) |
| **Threads** | ~11 | ~7-15+ | ~3 |
| **Erreurs** | Codes retour + side effects | `Result<T>` monadique | `bool` + `lastError()` |
| **Driver** | Monolithique | ITransport + MockTransport | TCP direct, pas de mock |
| **Test driver** | Contrôleur réel | MockTransport dans le binaire | xmul (émulateur externe) |
| **Cinématique** | DH + plugins analytiques | DH + plugins | DH d'abord, analytique + tard |
| **Lua** | Plugin LanguageEngine | sol2 sandboxé | sol2 sandboxé |
| **Web** | Mongoose + SPA JS | CivetWeb + REST API | CivetWeb (futur) |
| **ZeroMQ** | Oui (REP/PUB/DEALER) | Non (prévu, pas fait) | Non (futur) |
| **Couches d'indirection** | ~8 | ~12 | ≤4 |
